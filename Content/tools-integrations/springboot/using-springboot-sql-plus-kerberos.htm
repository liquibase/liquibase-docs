Using and Spring Boot with SQL Plus and Kerberos Authentication

The Spring Boot integration supports SQL Plus using the runWith=sqlplus s. To run the runWith types of s, requires the authentication mechanism that allows running the native executor without specifying credentials, such as Kerberos.

For now, the runWith=sqlplus is not available if you connect to your database using the following authentication mechanisms:

  • JNDI Datasource
  • External spring.datasource.* properties

However, the authentication mechanisms do not affect s which do not use the runWith=sqlplus .

Prerequisites

Ensure you have configured the following parts:

  • with Oracle
  • with Spring Boot

If you do not have the needed configuration, follow Using [%=General.Liquibase%] with Oracle and Using [%=General.Liquibase%] with Spring Boot by replacing the values from the documentation with your values.

Running Spring Boot and SQL Plus with Kerberos

  1. Configure and Kerberos.

    See Connecting to an Oracle Database with [%=General.Liquibase%] via Kerberos and Active Directory.

  2. Specify how can find SQL Plus by adding SQL Plus to your PATH. Alternatively, pass its location in the liquibase.sqlplus.conf file or from the command prompt using the --sqlplus-path parameter during runtime.

  3. Specify the runWith to the needed s in the file that you use. For more information, see Use SQL Plus and runWith on Oracle.

  4. Run the update command:

    5. If you have specified your file as an environment variable or in the properties file, you can skip passing it at the command prompt.

    liquibase update --changelog-file=<your changelog file>
