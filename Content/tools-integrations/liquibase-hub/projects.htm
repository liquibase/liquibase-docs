Projects

A Project is a structured collection of the resources used by or 4.1 and later versions. An Organization can contain many Projects that include a list of your Targets, s with their s, and the Operations that were run on the Project's database.

A app user with a valid API Key can register a to a Project. Registering a enables the operations such as update, rollback, changelog-sync, and drop-all to be recorded and available for review as Operations reports in . For more information, see [%=General.Liquibase%] Hub Operations.

All members of an Organization can create Projects, register their to the Organization's Project, and send and view Operation data to these Projects. An Organization Admin can manage access to Projects in the Organization by adding or removing members from the Organization.

Targets in the tracking table

Database schema changes to a specific target are stored in the local tracking table that exists in that target, which is synchronized with its counterpart when working with a registered . You cannot transfer target's data between Projects or Organizations, as it is automatically captured when you report commands to .

A Project can have multiple targets. However, do not share the same local table for two Organizations and Projects. The table is a core element tracked by local usage, which is synchronized and reported on at . It is best practice to register the s to the same Project if your workflow shares the same table.
