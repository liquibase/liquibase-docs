is a secure portal that provides real-time monitoring, reports, and insights into your team's database release automation. It provides centralized visibility into distributed operations performed against multiple environments.

The CLI app sends information to the portal when you run commands that make changes to your database.

Install

If you have already installed , skip this step.

Click Download and run the installer.

automatically detects your operating system and specific installation files to download. To use another operating system, go to the Download page. If you encounter any issues, check the Installation Troubleshooting page.

If you are new to , see Getting Started with [%=General.Liquibase%].

Configure

From the command prompt or terminal, run the liquibase init hub command with the token you see in the dialog and follow the CLI instructions depending on the database:

  • To use the H2 database that is included with the installation files, enter Y for the default settings.
  • To use your database, enter C and customize the username, password, and database URL parameters. After running the command, check or add a database driver.

The command connects your local activity to with an API key and set up the environment. If you are new to , the init hub command creates configuration files for you.

If you encounter issues with a token or environment setup, configure manually with the register-changelog command.

Alternatively, you can auto-register for when you run the update command. See [%=General.Liquibase%] Hub Auto-Registration and Report Links.

Deploy your first change

  • Navigate to the registered project directory and run the following command:

    • To use the in-memory H2 database, open a new terminal window and run the liquibase init start-h2 command before running the update command.

    liquibase update
  • Open your report to see the update details.
