Getting Started with

is a secure portal that provides the real-time monitoring, reports, organization, and insights into your team's database release automation. It provides centralized visibility into distributed operations performed against the multiple environments.

The CLI app sends information to the portal when you run the commands that make changes to your database. For example, update, changeLogSync, rollback, or dropAll.

If you use Maven, refer to the registerChangeLog and syncHub documentation to configure the Maven pom.xml file and another required properties.

To set up your account, follow these steps:

Step 1: Sign up for an account

  1. Go to the portal.
  2. Sign up for an account.
  3. Check your email and follow the link to activate an account.

Step 2: Add a API key to the file

  1. Sign in to your account and go to Settings > API Keys by selecting the settings icon on the left side of the page.
  2. Copy the API key, which connects the information generated by your s and other operations to your Projects. The API key is a new concept for organizing your work, and it is unique for each user. will only be in sync with your actions if the API key is added.
  3. Add your API key to the file as follows: liquibase.hub.ApiKey: <addYourHubApiKeyHere>.
# liquibase.properties
changeLogFile: ../path/to/file/dbchangelog.xml
driver: oracle.jdbc.OracleDriver
url: jdbc:oracle:thing:@192.168.0.22:1521/orcl
username: PRO
password: password
referenceDriver: oracle.jdbc.oracleDriver
referenceurl: jdbc:oracle:thin:@192.168.0.22:1521/orcl
referencePassword: password
liquibaseProLicenseKey: <addYourProLicenseKeyHere>
classpath: ../path/to/file/ojdbc6-11.2.0.3.0.jar
# Add Your First Liquibase Hub API Key
liquibase.hub.ApiKey: <addYourHubApiKeyHere>

Alternatively, you can pass your API key as a runtime argument and run your commands as usual or you can specify it in your JAVA_OPTS Environment Variable as -Dliquibase.hub.apiKey. To specify JAVA_OPTS on Linux or MacOS, enter JAVA_OPTS="-Dliquibase.hub.apiKey=<myapikey>". On Windows, enter set JAVA_OPTS="-Dliquibase.hub.apiKey=<myapikey>". It is a best practice to add the API key to the file or your build files instead of entering it manually every time.

java -jar liquibase.jar \
--driver=oracle.jdbc.Oracleoriver \

You can also set the level of data you want to send to but this process is optional:

liquibase.hub.mode:[all|meta|off]
  • all sends all data, including the body, generated SQL, and logs (default)
  • meta does not send the body, generated SQL, or operation-event and change-event logs
  • off sends no data to

Step 3: Run the registerChangeLog command

  1. Run the registerChangeLog command to connect an XML or formatted SQL to a specific Project. Also, you can use JSON and YAML formats (since 4.2.1). relies on a new unique identifier in your called changeLogId, which is automatically added by the registerChangeLog command to provide insights, real-time monitoring, and reports.

Register only the master without registering the nested s, if you use this best-practices structure. You can register a to an existing project or create a new one from the CLI as well.

  1. Once the changelogId identifier is added to your , ensure to commit the to source control, so all team members' changes are recorded in .

Step 4: Run your regular commands

Now, just use or , and will track the following commands:

  • update
  • updateCount
  • updateToTag
  • rollback
  • rollbackCount
  • rollbackOneChangeSet
  • rollbackOneUpdate
  • rollbackToDate
  • changelogSync
  • dropAll

Each command will automatically synchronize the local table with , but you can invoke this synchronization manually by using the new syncHub command.

So long as you have a registered , a valid API key, and hub.mode=[all|meta], you can use to see what s passed and what failed in your update, rollback, changelogSync, or dropAll operations. You can see these Operation reports filtered to a specific Project, database target, or by dates and users.
