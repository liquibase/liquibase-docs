Compatible Native Executors: SQL Plus, SQLCMD, and PSQL Integrations

There are three native executors that can be used with Liquibase Pro. One for Oracle databases, one for SQL Server databases, and another for PostgreSQL databases:

  • For Oracle, the native executor uses sqlplus which is Oracle's native database client.
  • For SQL Server, the native executor uses sqlcmd which is SQL Server's native database client.
  • For PostgreSQL, the native executor uses psql which is PostgreSQL's native database client.

Database native executors are not shipped with Liquibase and would need to be installed.

Using the runWith Attribute

Liquibase decides whether to use the native executor based on the presence of the runWith attribute. This attribute needs to be applied to each changeset that will be deployed with the native executor. When this attribute is not present, Liquibase will use the JDBC driver to deploy the changeset:

The runWith attribute works the same as sqlplus for sqlcmd (runwith:sqlcmd) and psql (runwith:psql).

There are three steps in setting up Liquibase deployments with a native executor:

  1. Install the native executor on your client machine.
  2. Configure the native executor in Liquibase.
  3. Add the runWith attribute to the necessary changesets.

Configure the Native Executor in Liquibase

You can configure Oracle's sqlplus, SQL Server's sqlcmd, or PostgreSQL's psql using one of the following methods:

  • A liquibase.sqlplus.conf file for sqlplus, liquibase.sqlcmd.conf file for sqlcmd, and a liquibase.psql.conf for psql.
  • A Liquibase properties file.
  • Using Liquibase environment variables.
  • Using Liquibase CLI arguments.

A typical configuration in a liquibase.sqlplus.conf or the Liquibase properties file may look like this:

# Sample linux path
liquibase.sqlplus.path=/apps/app/12.2.0.1.0/oracle/product/12.2.0.1.0/client_1/bin/sqlplus

# Sample windows path
# liquibase.sqlplus.path=c:\\oracle\\product\\11.2.0\\client_1\\bin\\sqlplus.exe

# A valid timeout value for the execution of the SQLPLUS tool
liquibase.sqlplus.timeout=-1

# Flag to indicate whether or not to keep the temporary SQL file after exection of SQLPLUS.
# True = keep False = delete (default)
liquibase.sqlplus.keep.temp=true

A typical configuration using environment variables may look like this: 

# Sample linux path
			LIQUIBASE_SQLPLUS_PATH=/apps/app/12.2.0.1.0/oracle/product/12.2.0.1.0/client_1/bin/sqlplus

			# Sample windows path
			# LIQUIBASE_SQLPLUS_PATH=c:\\oracle\\product\\11.2.0\\client_1\\bin\\sqlplus.exe

			# A valid timeout value for the execution of the SQLPLUS tool
			LIQUIBASE_SQLPLUS_TIMEOUT=-1

			# Flag to indicate whether or not to keep the temporary SQL file after exection of SQLPLUS.
			# True = keep False = delete (default)
		LIQUIBASE_SQLPLUS_KEEP_TEMP=true
