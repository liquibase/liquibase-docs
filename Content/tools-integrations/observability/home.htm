Observability

Observability enables users to share encapsulated information about operations across teams with clear organization so users can make informed decisions that affect their database.

When operational data is plugged into an observability DevOps dashboard and reporting platform, it provides visibility into database change management and functionality to ensure peak performance and diagnose issues faster to reduce downtime.

Insights and actions powered by database observability can:

  • Reduce remediation time.
  • Identify drifts in a database state.
  • Enable trend analysis.

You can integrate operation data into third-party observability and analysis systems by enabling and configuring structured logs.

[%=General.StructuredLogging%] groups relevant pieces of information in machine-readable JSON formatted messages so operation data can be easily ingested by industry-standard observability and analysis tools.

When structured logs are enabled:

  • The same information in the default log is presented as a key:value pair.
  • Once structured logs are enabled, log entries will provide comprehensive details about each executed, including operation types, timestamps, command names, SQL statements, and outcomes.

For a list of endpoints in a structured log, see Structured Logging Keys.

Custom Log Data

You can also introduce your custom and private data into s using custom keys.

Custom keys are additional fields or attributes that users can define and include in their structured logs to provide more context or information about logged events. This enables dashboard queries and reports to tailor logging to an organization's observability needs.

To learn how to set up a custom log file, see Custom and Private Data in [%=General.StructuredLog%]s.

Operation Reports

Operation Reports let you generate reports of operations you perform on your database. These reports contain information about the status of your database and deployments and may help you resolve errors in your automation pipeline. You can generate the following reports:

  • Drift Report when you run the diff and diff-changelog commands
  • Update Report when you run an update family command
  • Rollback Report when you run a rollback family command
  • Checks Report when you run the checks run command
