Structured Logging Keys

uses a Mapped Diagnostic Context (MDC) to generate structured logs in a JSON format. The output is a nested group of key-value pairs. You can access data from your structured logs by calling on the keys described on this page.

System keys

The following keys can appear in all structured logs, regardless of which command the log originates from:

Key Description
class Java class associated with log event
level Level of detail of the output. See log-level.
liquibaseInternalCommand Name of the internal operation
liquibaseCommandName Name of the command entered in the CLI
liquibaseSystemName System name running
liquibaseSystemUser Name of user running
liquibaseTargetUrl URL of the target database associated with the log
liquibaseVersion Version of installed
message Short descriptor of log event
operationStart Timestamp of operation start
operationStop Timestamp of operation end
thread Thread being used
timestamp Timestamp of log event

Key Command Description
changelogFile update* The containing s being deployed
changesetComment update* Comment applied to
changesetLabel update* Label applied to
changesetOperationStart update* Timestamp of operation start. Repeats in every log message
changesetOperationStop update* Timestamp of operation end. Repeats in every log message
changesetSql update* Generated/executed SQL
changesetsUpdated update*

The s that were updated. Contains nested keys:

  • changesetCount
  • changeset
    • changesetAuthor
    • changesetFilepath
    • changesetId
    • changesetOutcome
    • deploymentId

commandLabelFilter update* Value of --label-filter

deploymentOutcome update* Outcome of the database deployment
deploymentOutcomeCount update* Integer
databaseChangelogSql update* SQL to update the table
databaseChangelogTableOutcome update* Outcome of update to the table
rollbackOnError update* Value of --rollback-on-error
updateSummary update*

Only if --show-summary=summary. Contains nested keys:

  • value
  • run

  • skipped

    • context
