Run commands on AWS

This guide covers how to run commands against the purchased license with AWS tools such as AWS Fargate, Amazon Elastic Container Service (ECS), Amazon Elastic Kubernetes Service (EKS), AWS CodePipeline, and AWS CloudFormation.

  1. Search and select in the AWS search bar search for the tool of your choice. In this example we are using Amazon Elastic Container Service (ECS).
  2. Navigate to the Task definitions section on the left side navigation. Task definitions are equivalent to commands, and all tested definitions are available in this window.
    1. Optional: Locate the full image URL in a task definition to extend the docker image
      This is where you may want to extend the docker image. To extend our docker image and add other databases for a specific task definition, you must use the image URL that displays the ECR repository. The steps below will show you how to locate this image URL.

      As of Liquibase 4.29.1 AWS does not support custom domain names for Amazon ECR.

    2. From the Task definitions section, select a task definition you want to extend the docker image of.

    3. Check the box next to the specific definition you want to revise.

    4. Click the Create new revision drop down menu and select Create new revision.

    5. From the Create new revision page you will see the image URL which is the ECR repository. This URL will be used when you extend the docker image.

  3. Run a task definition by selecting the radio button next to the desired definition. In this example we will use update-command.
  4. Select the Deploy button at the top of the screen.
  5. From the drop-down menu, select run task.
  6. Select the Compute options that you desire. In this example we will toggle Launch type on.
  7. Click the Create button.
  8. AWS will bring you to the tasks in your cluster. You will be able to see the task that is running in the Tasks tab below the Cluster Overview.
  9. Click on the task that is running in the Tasks tab.
  10. Click on the Logs tab to see all log events on any container you have access to.
