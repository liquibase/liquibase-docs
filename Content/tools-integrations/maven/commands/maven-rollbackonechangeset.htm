Maven rollbackOneChangeSet

rollbackOneChangeSet rolls back one non-sequential that is deployed to your database. It is only available for users.

Uses

rollbackOneChangeSet is typically used when you want to undo a change made to your database and revert it to a previous state without affecting any other changes made to your database.

While the Maven rollback goal reverts all the deployed s sequentially to a specified point, the rollbackOneChangeSet goal allows you to target a specific without impacting s that came before or after it.

The image shows s A through I, with I being the most recently deployed. As you can see, the rollbackOneChangeSet goal allows you to target B and revert it to its previous state without impacting the others.

The impacts of rollbackOneChangeSet

Using the rollbackOneChangeSet goal comes with risks of unintended consequences. For this reason, it is a best practice to run the Maven rollbackOneChangeSetSQL goal because it allows you to inspect the rollback SQL and search for any potential mistakes before you execute rollbackOneChangeSet.

Also, as the use of rollbackOneChangeSet comes with risks of unintended consequences, the goal requires a liquibase.force to specify that you intend to run the goal.

Maven configuration

Maven can be configured in multiple ways. One way is to define your configuration properties in your pom.xml file. To configure your pom.xml file, refer to Configuring [%=General.Liquibase%] [%=General.Param/Attr/Uppercased%]s in your Maven POM File. If pom.xml does not reference a proerties file, include <changeLogFile>specify your file here</changeLogFile> and <ProLicenseKey>specify your license key here</ProLicenseKey> to pom.xml.

Running the rollbackOneChangeSet Maven goal

Before running the rollbackOneChangeSet goal, gather the following information from your table:

  • The author of the you want to revert
  • The id of the you want to revert
  • The filename ( path) of the you want to revert

The liquibase.rollbackScript script is only needed if the rollback is not yet defined in the and if it is not a rollback that is automatically provided by .

Also, you can run the Maven history goal to see the history of all deployments against the configured database:

mvn liquibase:history

Next, run the rollbackOneChangeSet goal, with your information:

Running the rollbackOneChangeSet goal requires a Maven project to be implemented.

mvn liquibase:rollbackOneChangeSet
-Dliquibase.changeSetAuthor="LiquibaseProUser"
-Dliquibase.changeSetId="createProSchema"
-Dliquibase.changeSetPath="changelog.xml" -Dliquibase.force

Enter the name of the author, id, and the path with the filename that you want to use in place of LiquibaseProUser, createProSchema, and changelog.xml. For more information, run mvn liquibase:help -Ddetail=true -Dgoal=rollbackOneChangeSet.
