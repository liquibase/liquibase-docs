Maven deactivateChangeLog

The deactivateChangeLog goal removes the changelogID from your file so it stops sending reports to .

Uses

The deactivateChangeLog goal is typically used when:

  • You start using to keep a history of all your activity, but you have registered the wrong file and want to keep it from reporting the activity to .
  • You use a file that is being secluded or refactored into other s and do not want more than one source of s reporting to .

When you run the deactivateChangeLog goal, it modifies a specific file by removing changelogID to prevent it from sending the data. The goal differs from the liquibase.hub.mode=off property, which is set in the properties file or passed as a JAVA property and which prevents s from sending data.

The liquibase.hub.mode=off property will not report your activity to , regardless of a file, and will not ask to report the activity to when using a non-registered .

Maven configuration

Maven can be configured in multiple ways. One way is to define your configuration properties in your pom.xml file. To configure your pom.xml file, refer to Configuring [%=General.Liquibase%] [%=General.Param/Attr/Uppercased%]s in your Maven POM File.

Running the deactivateChangeLog Maven goal

Running the deactivateChangeLog goal requires a Maven project to be implemented. To run the deactivateChangeLog goal, include the file for the deactivation in the pom.xml or the properties file that pom.xml references. Specify the pom.xml property as liquibase.changeLogFile in the global properties section or as changeLogFile in the plugin configuration section.

Next, type the following in your goal prompt:

mvn liquibase:deactivateChangeLog

The changelogID is removed from the file that is specified in pom.xml or the properties file.

If the file is in source control, commit the newly deactivated .
