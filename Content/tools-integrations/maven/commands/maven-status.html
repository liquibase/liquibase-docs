Maven status

status produces a list of pending s with additional information that includes the id, author, and file path name. The status goal does not modify the database, it prints the output with s that need to be applied to the database.

Uses

status is typically used when s were added to a through source control by another developer. The goal confirms what has been deployed and what s are pending per author and corresponding ids.

Maven configuration

Maven can be configured in multiple ways. One way is to define your configuration properties in your pom.xml file. To configure your pom.xml file, refer to Configuring [%=General.Liquibase%] [%=General.Param/Attr/Uppercased%]s in your Maven POM File.

Running the status Maven goal

Running the status goal requires a Maven project to be implemented.

To run the goal, type the following in your command prompt:

mvn liquibase:status

To see more details in the output, add the liquibase.verbose to the goal: mvn liquibase:status -Dliquibase.verbose.
