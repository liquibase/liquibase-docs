TableCommentCheck

The TableCommentCheck subcommand allows the user to flag any table that does not have a comment. Note that this is not a check for a  changelog comment.

  • Subcommands require specific parameters noted in the table.
  • Customizations to checks configuration are stored in the liquibase.checks-settings file.

Uses

You can use the subcommand to check for the presence of a comment within a table described in a changelog. Comments explain the purpose of a table for anyone viewing the table.



Run the TableCommentCheck subcommand

To run the TableCommentCheck subcommand, ensure you have a license, the checks-scope property specified as Database in the Default Properties file, a valid URL property, and a url property pointing to a live JDBC connection or a snapshot. Once those items are established, check the table for comments. Enable the TableCommentCheck check by running the following command in the CLI: 
liquibase checks enable --check-name=TableCommentCheck
