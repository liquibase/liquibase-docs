history

The history command lists all deployed s and their deploymentIds. The history command does not modify the database.

Uses

The history command is typically used when you want to inspect a particular group of changes to ensure that they have been applied to the database.

Syntax

To run the history command, specify the driver, classpath, and URL in the properties file. For more information, see Create and Configure a [%=General.liquiPropFile%] File. You can also specify these properties in your command line.

Then run the history command:

liquibase history

Parameters

  • CLI
  • Flow
  • liquibase.properties
  • JAVA_OPTS
  • Environment variable
Output
