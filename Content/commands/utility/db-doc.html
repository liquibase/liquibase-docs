db-doc

The db-doc command generates documentation in a Javadoc format based on the existing database and s.

Uses

The db-doc command is typically used to generate database documentation with the change information stored in the s and the existing database. It captures the current state of your database including everything that has been performed against it.

Syntax

To run the db-doc command, specify the driver, classpath, and URL in your properties file. For more information, see Create and Configure a [%=General.liquiPropFile%] File. You can also specify these properties from the command line.

Then run the db-doc command:

liquibase db-doc --output-directory=exampleDirectory --changelog-file=example-changelog.xml

The --output-directory=exampleDirectory syntax was added in 4.4. If you use an older version, specify your output directory as a positional argument: <command> exampleDirectory. It is best practice to specify a new empty directory because the db-doc command creates its own directories, the CSS file, and multiple HTML files.
