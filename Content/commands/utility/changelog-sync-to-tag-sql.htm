changelog-sync-to-tag-sql

changelog-sync-to-tag-sql is a helper command that produces the raw SQL that Liquibase would run when using the changelog-sync-to-tag command to mark all undeployed s associated with the specified tag as executed in your database.

Uses

The changelog-sync-to-tag-sql command is typically used when you want to inspect the SQL Liquibase will use to baseline a new database environment.

It is best practice to inspect any SQL that Liquibase would run when using the changelog-sync-to-tag command so you can review any changes the command would make to your database before running it. The changelog-sync-to-tag command marks all s starting with the first at the top of the file and moving down to the up to and including the tag.

If you donâ€™t have any tag specified in your file, add it by using the tagDatabase as follows:

<changeSet id="13.1" author="liquibase">
    <tagDatabase tag="version_2.0"/>
</changeSet>

When you run changelog-sync-to-tag-sql, check whether you specified the correct tag and which s were affected. (If you specify a tag that Liquibase cannot find in your , the changelog-sync-to-tag command adds new rows in the for all changes.)

Syntax

Before running the changelog-sync-to-tag-sql command, specify the driver, classpath, and URL in the Liquibase properties file. For more information, see Create and Configure a [%=General.liquiPropFile%] File. You can also specify these properties in your command line.

Next, run the changelog-sync-to-tag-sql command:

liquibase changelog-sync-to-tag-sql --tag=myTag --changelog-file=example-changelog.xml

Parameters

Output

When successful, the changelog-sync-to-tag-sql command produces the following output:

-- Ran at: 5/13/22, 10:17 AM
-- Against: DBUSER@jdbc:h2:tcp://localhost:9090/mem:dev
-- Liquibase version: 4.9.1
-- *********************************************************************

-- Lock Database
UPDATE PUBLIC.DATABASECHANGELOGLOCK SET LOCKED = TRUE, LOCKEDBY = 'DESKTOP-SQVSSV2 (192.168.1.179)', LOCKGRANTED = NOW() WHERE ID = 1 AND LOCKED = FALSE;

-- Release Database Lock
UPDATE PUBLIC.DATABASECHANGELOGLOCK SET LOCKED = FALSE, LOCKEDBY = NULL, LOCKGRANTED = NULL WHERE ID = 1;
