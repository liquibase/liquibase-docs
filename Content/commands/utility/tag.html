tag

The tag command marks the current database state so you can roll back changes in the future.

If you want to apply a tag to your database within your changelog, write a changeset containing the tagDatabase . To check whether a tag exists, use the tag-exists command.

Uses

The tag command is typically used to mark the current database state, version, release, or any other information by adding the tag to the last row in the table. After setting the tag, you can use the rollback command to roll back all changes up to that tag.

The image below shows the table structure. Running the command tag version1 applies the tag version1 to the last row in the table. If you run rollback version1 command, it will roll back only createTable C value.

It is best practice to tag your changesets before running any other commands against your database.

Syntax

To run the tag command, specify the driver, classpath, and URL in the Liquibase properties file. For more information, see Create and Configure a [%=General.liquiPropFile%] File. You can also specify these properties in your command line.

Then run the tag command:

liquibase tag --tag=myTag
