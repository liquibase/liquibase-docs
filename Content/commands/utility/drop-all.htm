drop-all

Syntax

To run the drop-all command, specify the driver, classpath, and URL in the Liquibase properties file. For more information, see Create and Configure a [%=General.liquiPropFile%] File. You can also specify these properties in your command line.

Then run the drop-all command:

liquibase drop-all

If you use Liquibase 4.27.0 and later, it is a best practice to specify liquibase.command.dropAll.requireForce or LIQUIBASE_COMMAND_DROP_ALL_REQUIRE_FORCE in your file or as an environment variable. This makes it mandatory for developers to specify --force whenever they run drop-all in the CLI:

liquibase drop-all --force

Impact

Using the drop-all command comes with risks of unintended consequences. Because of this, in Liquibase 4.27.0 and later, the command lets you use --require-force to require developers to specify a --force flag to indicate that they intend to run it.

Look for potential unintended consequences before using this command.

Output

When successful, the drop-all command produces the following output:

Liquibase Pro 3.9.0  licensed to Liquibase Pro Customer until Tue Sep 22 19:00:00 CDT 2020
All objects dropped from MVN_UAT@jdbc:oracle:thin:@support.datical.net:1521:ORCL
Liquibase command 'drop-all' was executed successfully.
