calculate-checksum

The calculate-checksum command calculates and prints a checksum for a in your .

Uses

The calculate-checksum command is typically used to compute an MD5 checksum, which serves as a unique identifier for the . As a result, you can see whether the has been changed and whether it has to be deployed differently now.

When running the calculate-checksum command, the table calculates an MD5 checksum for each entry based on the SQL script of the . This checksum helps detect differences between the s you want to deploy and the s that have already been run against the database.

The MD5SUM column in the table contains a checksum of the and any change made in the will result in a different checksum.

Syntax

To run the calculate-checksum command, specify the driver, classpath, and URL in the properties file or in your command line. For more information, see Create and Configure a [%=General.liquiPropFile%] File.

When you run calculate-checksum, you must identify which to target by specifying its filepath, ID, and author in the following format: filepath::id::author. The filepath refers to the path from the current working directory to the that your is in. For example:

liquibase calculate-checksum --changeset-identifier=com/example/example-changelog.xml::1::adrian

If you want to run this command on a that uses the logicalFilePath , specify the logical filepath instead.

The --changeset-identifier=value syntax was added in 4.4. If you use an older version, specify your identifier as a positional argument: <command> value.
