update-count-sql

The update-count-sql command is a helper command that inspects the SQL will run while using the update-count command.

Uses

The update-count-sql command is used to inspect the raw SQL before running the update-count command, so you can correct any issues that may arise before running the command. uses the raw SQL to apply a specified number of database changes you have added to the file.

Syntax

To run the update-count-sql command, specify the driver, classpath, and URL in the properties file. For more information, see Create and Configure a [%=General.liquiPropFile%] File. You can also specify properties from the command line.

Then run the update-count-sql command:

liquibase update-count-sql --count=2 --changelog-file=example-changelog.xml

--show-summary=[summary | verbose]

Produces a summary list or a verbose summary list of any changesets skipped and why they were skipped

There can be multiple reasons listed. Simultaneously, only one reason may display in the summary. This indicates that Liquibase halted trying to deploy this changeset for this reason, although additional reasons might also exist. If Liquibase cannot resolve that halting reason, the changeset is not evaluated for other possible reasons that are not currently displayed.
