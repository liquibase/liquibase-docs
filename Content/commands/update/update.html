update

The update command deploys any changes that are in the file and that have not been deployed to your database yet.

Uses

The update command is typically used to apply database changes that are specified in the file to your database.

When you run the update command, Liquibase sequentially reads s in the file, then it compares the unique identifiers of id, author, and path to filename to the values stored in the table.

  • If the unique identifiers do not exist, Liquibase will apply the to the database.
  • If the unique identifiers exist, the MD5Sum of the is compared to the one in the database.
  • If they are different, Liquibase will produce an error message that someone has changed it unexpectedly. However, if the status of the runOnChange or runAlways is set to TRUE, Liquibase will re-apply the .

Syntax

To run the update command, specify the driver, classpath, and URL in the Liquibase properties file. For more information, see Create and Configure a [%=General.liquiPropFile%] File. You can also specify these properties in your command line.

Then run the update command:

liquibase update --changelog-file=example-changelog.xml

Parameters

  • Flow
  • liquibase.properties
  • JAVA_OPTS
