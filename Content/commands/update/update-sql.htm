update-sql

The update-sql command is a helper command that allows you to inspect the SQL will run while using the update command.

Uses

The update-sql command is used when you want to inspect the raw SQL before running the update command, so you can correct any issues that may arise before running the command. uses the raw SQL to apply database changes you have added to the file.

Syntax

To run the update-sql command, specify the driver, classpath, and URL in the properties file. For more information, see Create and Configure a [%=General.liquiPropFile%] File. You can also specify properties from the command line.

Run the update-sql command:

liquibase update-sql --changelog-file=example-changelog.xml

Parameters

  • CLI
  • Flow
  • liquibase.properties
  • JAVA_OPTS
  • Environment variable

* checks the and any nested s for the definitions of the s to update.

Precondition behavior

Preconditions are tags you add to your or individual s to control the execution of an update based on the state of the database. You can specify the following s to control whether the update-sql command runs preconditions:

  • onSqlOutput: controls how preconditions are evaluated with the update-sql command for XML, YAML, and JSON s.
  • onUpdateSql: controls how preconditions are evaluated with the update-sql command for formatted SQL s.
