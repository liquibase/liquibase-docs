update-one-changeset-sql

The update-one-changeset-sql command is a helper command that allows you to inspect the SQL will run to deploy the specified in the update-one-changeset command. It is only available for users.

Uses

The update-one-changeset-sql command is useful when you want to inspect the raw SQL that uses to deploy your when you run the update-one-changeset command, so that you don't unintentionally make a mistake.

Syntax

Before running the update-one-changeset-sql command, gather the following information from your :

Then run the update-one-changeset-sql command, with your information:

liquibase update-one-changeset-sql --changelog-file=example-changelog.xml --changeset-id=2 --changeset-author=anotherdev --changeset-path=example-changelog.xml

For more command-specific help, type liquibase update-one-changeset-sql --help into the command prompt.

--show-summary=[summary | verbose]

Produces a summary list or a verbose summary list of any changesets skipped and why they were skipped

There can be multiple reasons listed. Simultaneously, only one reason may display in the summary. This indicates that Liquibase halted trying to deploy this changeset for this reason, although additional reasons might also exist. If Liquibase cannot resolve that halting reason, the changeset is not evaluated for other possible reasons that are not currently displayed.

 Optional
--username Optional
Output

When successful, the update-one-changeset-sql command produces the following output:

$ liquibase update-one-changeset-sql --changeset-id=2 --changeset-author=anotherdev --changeset-path=example-changelog.sql

-- Initialize Database Lock Table
DELETE FROM DATABASECHANGELOGLOCK;
