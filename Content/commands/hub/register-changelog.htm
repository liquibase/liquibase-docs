register-changelog

The register-changelog command connects your local activity to a specific Project. By registering your , this activity will be visible only to one Project within one Organization in .

Uses

Use the register-changelog command to connect an XML, SQL, JSON, or YAML to your Project.

JSON and YAML formats are available in 4.2.1 and later versions.

The command assigns the changeLogId to the file that you register. The changeLogId is a unique identifier in your , which is automatically added by the register-changelog command to provide real-time monitoring and reports.

You can connect the needed file with your existing project or create a new project.

If you follow a root structure, register only the root without registering the nested s. For more information about the structure, see Best Practices.

Running the register-changelog command

To run the register-changelog command, specify the following parameters in the properties file, environment variables, or the command prompt while running the command:

  • Driver, URL, and user authentication information such as username and password
  • API key
  • file

Run the register-changelog command:

liquibase register-changelog --changelog-file=example-changelog.xml

Non-interactive register-changelog

To run the register-changelog in the non-interactive mode, specify the --hubProjectId, which is located in Projects > View Details for the required project > Operations or any other tab of the project.

If you specify both and hubProjectId, the command will register the to the specified project without showing the interactive prompt: 

liquibase register-changelog --changelog-file=example-changelog.xml --hubProjectId=myProjectId

The Maven register-changelog goal also runs in the non-interactive mode. For more information, see Maven registerChangeLog.
