deactivate-changelog

The deactivate-changelog command removes the changelogID from your file so it stops sending reports to .

Uses

Use the deactivate-changelog command when:

  • You start using to keep a history of all your activity, but you have registered the wrong file and want to keep it from reporting the activity to .
  • You use a file that is being secluded or refactored into other s and do not want more than one source of s reporting to .

When you run the deactivate-changelog command, it modifies a specific file by removing changelogID to prevent it from sending the data. The command differs from the liquibase.hub.mode=off property, which is set in the properties file or passed as a JAVA property and prevents any from sending data.

The liquibase.hub.mode=off property will not report your activity to , regardless of a file, and will not ask to report the activity to when using a non-registered .

Syntax

Before running the deactivate-changelog command, specify the driver, classpath, and URL in the properties file. For more information, see Create and Configure a [%=General.liquiPropFile%] File. Otherwise, specify the properties at the command prompt.

Run the deactivate-changelog command:

liquibase deactivate-changelog --changelog-file=example-changelog.xml

This removes the ID from your . If the file is in source control, commit the newly deactivated .
