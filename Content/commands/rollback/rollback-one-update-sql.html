rollback-one-update-sql

The rollback-one-update-sql command is a helper command that allows you to inspect the SQL will run to revert all associated with the deploymentID specified in the rollback-one-update command. It is only available for users.

Uses

The rollback-one-update-sql command is typically used when you want inspect the raw SQL that uses to revert all s associated with a specified deploymentId when you run the rollback-one-update command, so you don't unintentionally make a mistake.

Syntax

Before running the rollback-one-update-sql command, you can get the optional deploymentId of the group you want to revert by running the history command.

Then run the rollback-one-update-sql command, with your information:

liquibase rollback-one-update-sql --changelog-file=example-changelog.xml --deploymentId=2068379006

Unlike the rollback-one-update command, there is no impact to the table.

For more command-specific help, type liquibase rollback-one-update-sql --help into the command prompt.

Parameters

  • CLI
  • Flow
  • liquibase.properties
  • JAVA_OPTS
  • Environment variable

* checks the and any nested s for the definitions of the to roll back.
