calculateCheckSum <id> command

The calculateCheckSum <id> command calculates and prints a checksum for the with the specified id in the following format: filepath::id::author.

Uses

The calculateCheckSum <id> command is typically used to compute an MD5 checksum, which serves as a unique identifier for the . As a result, you can see whether the has been changed and whether it has to be deployed differently now.

When running the calculateCheckSum <id> command, the table calculates an MD5 checksum for each entry based on the SQL script of the . This checksum helps detect differences between the s you want to deploy and the s that have already been run against the database.

The MD5SUM column in the table contains a checksum of the and any change made in the will result in a different checksum.

Running the calculateCheckSum <id> command

To run the calculateCheckSum <id> command, you need to specify your driver, class path, URL, and user authentication information in your liquibase.properties file. For more information, see Creating and configuring a liquibase.properties file. You can also specify these properties in your command line.

Then run the calculateCheckSum <id> command:

liquibase calculateCheckSum changelog.oracle.sql::myID_123::Steve

Enter the name of your , id, and author name that you want to use in place of changelog.oracle.sql, myID_123, and Steve. If you use the .sql file, include your database type in place of oracle.
