RegexNegLookBehind

The RegexNegLookBehind check is a pre-written Regex quality check that allows to look for a positive pattern based on two user-provided inputs. This check is pre-configured so users do not have to hand write regex's to look forward in their .

This check triggers when the user-supplied regular expression for a primary match is not preceded by a user-supplied regex 'lookbehind' match.

Uses

The RegexNegLookBehind check can be used to confirm conditions within the . This check allows you to check for specific patterns in your and then produce a logged record of structure when certain conditions are met. When it is executed successfully the checks message is logged to the console and logs as proof that s conform to specific standards.

Run the RegexNegLookBehind check

To run the RegexNegLookBehind check, ensure you have a Liquibase Pro license and the checks-scope property includes changelog. The checks-scope parameter is set in the default properties file, environment variable, or any standard method.

LookBehind Logic:
If regex pattern X exists then Pattern A must not exist before pattern X.

In this example, you are ensuring that all instances of Pattern A do not exist before pattern X. When this check runs successfully, a log will be created that displays all instances of Pattern A that do not exist before pattern X in your .

Enable the RegexNegLookBehind check by running the following command in the CLI:
liquibase checks enable --check-name=regexNegLookBehind
