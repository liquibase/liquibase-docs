Changelog and Database Quality Checks

Dual-scoped changelog and database quality checks allow you to confirm the presence of specific objects and conditions in both the and database to ensure their integrity.

To execute these quality checks, ensure that --checks-scope contains changelog or database.

For more information about using quality checks, see Quality Checks. This list excludes checks which have only a single scope. For a list of all checks, see List of Quality Checks.
