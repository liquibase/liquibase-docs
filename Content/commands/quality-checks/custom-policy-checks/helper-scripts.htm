Custom Policy Check Helper Scripts

Helper Scripts are Python scripts that provide an API that allows you to access objects or functionality with Custom Policy Checks. When helper scripts are enabled within a custom policy check, the check goes through all the s or database objects that have not deployed and looks for the specific content set in the custom policy check. will then decide if the check fires or not depending on what is specified with the helper script. also provides a python package liquibase-checks-python to assist with in-IDE documentation and auto complete. These helper scripts and Python package significantly reduce the time it will take to find the information you are looking for.

The Checks extension, which includes all Custom Policy Check functionality, is available to Pro 4.29+ users.

Compatible Versions
Python 3.10.14
GraalPy 24.0.0

Install the Python installer package (PIP)

provides a Python package called liquibase-checks-python to assist with in-IDE documentation and auto complete with your helper scripts. Install our Python package by following these instructions.

Step-by-step

  1. Create a virtual environment in your directory that contains the custom policy check script by running the following in the CLI:

    python -m venv <path to your venv>

  2. Install the Python package by running pip install liquibase-checks-python in the CLI.
  3. In your custom policy check file, replace import liquibase_utilities with from liquibase_checks_python import liquibase_utilities in your Python script.
  4. from liquibase_checks_python import liquibase_utilities as lb # this is where we are importing our liquibase_utilities helper script provided by the liquibase-checks-python package import sys # useful for indicating a check has been triggered obj = lb.get_database_object() # this gets the database object the liquibase policy check is examining status = lb.get_status() # this gets the status object of the liquibase check, used for reporting status and messages from the custom policy check if lb.is_table(obj): # function provided from liquibase utilities status.fired = True # indicates the custom check has been triggered status.message = "No tables allowed!" # set the message of the custom check, which liquibase will return sys.exit(1) # exit from the check

  5. You have successfully activated the Python package. Next you will need to activate the helper scripts to run them.

Virtual Environment Options

A virtual environment and using the GraalPy CLI tool are optional steps and are not required to use Custom Policy Checks.

You can use the provided virtual environment that is part of the release or your own Python virtual environment. If you need to utilize your own virtual environment you will need to have Python 3.10 installed on your system. This virtual environment is stored in a directory on your local file system.

custom policy checks use GraalPy to execute your custom checks. We recommend downloading and using the GraalPy python runtime if you are working in a virtual environment. The pip package installer is available when using a virtual environment. The GraalPy implementation of pip may choose package versions other than the latest when it ships patches to improve performance.

Import and access helper scripts

Step-by-step

If you want the custom policy check helper scripts with in-IDE documentation and auto-complete you must complete the Install the Python installer package (PIP) steps first. If you want the custom policy check helper scripts without those features, skip that section and follow the Import and access helper script steps.

  1. Navigate to your Custom Policy Check Python script that you want to implement helper scripts in.
  2. Import the utilities helper scripts by placing import liquibase_utilities within the Python script.
    Now the liquibase_utilities Python package that includes the helper scripts are accessible to .
    import sys import liquibase_utilities
  3. Next you will include a specific helper script function in your Custom Policy Check Python script.
    In this example we will include logger = liquibase_utilities.get_logger to pull the log.
  4. ### ### Retrieve log handler ### Ex. liquibase_logger.info(message) ### liquibase_logger = liquibase_utilities.get_logger() Run this command to enable your custom policy check in the CLI.
    liquibase checks run --check-name <custom_policy_check_example> --checks-scripts-enabled
    When the custom check runs with these specifications it will retrieve any content it finds based on the packages and the helper script functions you chose to include and will display it in the CLI or in a report.
    In this example, will produce the log information when you call on this Custom Policy Check because it includes logger = liquibase_utilities.get_logger.
