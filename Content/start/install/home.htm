Install Liquibase

Software installation

You can install Liquibase on Windows, MacOS, and Linux/Unix systems that meet the hardware Requirements:

  • Install [%=General.Liquibase%] on Windows
  • Install [%=General.Liquibase%] on macOS
  • Install [%=General.Liquibase%] on Linux/Unix
    • Install [%=General.Liquibase%] on Linux with Debian/Ubuntu
    • Install [%=General.Liquibase%] on Linux with Red Hat/CentOS
  • Install [%=General.Liquibase%] with Ansible
  • Install [%=General.Liquibase%] with Chocolatey
  • Install [%=General.Liquibase%] with SDKMAN

Software configuration

The Liquibase CLI ships with drivers for many popular database systems in the liquibase/internal/lib directory, including:

  • DB2
  • H2
  • HyperSQL
  • Firebird
  • MariaDB
  • MS SqlServer
  • Oracle
  • PostgreSQL
  • Snowflake
  • SQLite

If you are using a different database system or need to run a different version of the driver, you can download the appropriate driver from your vendor, and install it in one of two places:

  • The lib directory of your Liquibase installation – Drivers added here will always be used by Liquibase
  • A liquibase_libs directory of your current working directory – Drivers added here will only be used when you are in this working directory

If you are using any Liquibase extensions, they can be added to the same directories.

Manual library management

Liquibase comes with two JAR files in liquibase/internal/lib serving as primary libraries:

  • liquibase-core-.jar
  • liquibase-commercial-.jar.

You need a commercial license key to access functionality for and other commercial add-ons. If you aren't using the liquibase-commercial JAR and want to exclude it from your project, navigate to liquibase/internal/lib in your command line and manually remove the file:

Linux/macOS:

rm liquibase-commercial-.jar

Windows:

del liquibase-commercial-.jar

For more information about individual Liquibase files, see [%=General.Liquibase%] Installation Files.

Software upgrades

Liquibase regularly releases new versions with additional features, which you can read about in the [%=General.Liquibase%] Release Notes. To learn best practices about upgrading to newer versions of Liquibase, see Upgrade [%=General.Liquibase%].
