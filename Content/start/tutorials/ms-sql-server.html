Using with Microsoft SQL Server

Microsoft SQL Server (MSSQL) is a relational database management system. Microsoft SQL Server supports different editions and components that accommodate unique performance and depend on your specific requirements.

For more information, see the Microsoft SQL Server documentation page.

You can also use:

  • Amazon RDS for SQL Server
  • Microsoft Azure SQL Database
  • Microsoft Azure SQL Managed Instance
  • SQL Server on Google Cloud

Prerequisites

Install drivers

CLI users

To use and Microsoft SQL Server, you need the JDBC driver JAR file (Maven download).

<dependency>
    <groupId>com.microsoft.sqlserver</groupId>
    <artifactId>mssql-jdbc</artifactId>
    <version>12.3.1.jre8</version>
</dependency>

Microsoft Azure SQL Managed Instance

If you use a Microsoft Azure SQL Managed Instance and want to authenticate using a Managed Identity Service Principal connection, you need several dependencies to connect to .

In 4.23.1+, you can install a single JAR file liquibase-azure-deps-.jar containing all dependencies instead of installing each JAR manually. Download the JAR with your version. Place your JAR file in the liquibase/lib directory.

For more information, see Connect using Azure Active Directory authentication.

Configure connection

  1. Ensure your Microsoft SQL Server is configured. You can check the status by using one of a management tool or by running the ping command. Also, you can use the sqlcmd utility and run sqlcmd -SmyServer\instanceName.

    2. Replace myServer\instanceName with the name of the computer and the instance of SQL Server that you want to connect to.

    2. url: jdbc:sqlserver://<host>:<port>;databaseName=<dbname>

    Depending on the configuration you use, your URL format may be different. For more information, see Building the Connection URL and Setting the connection properties.
