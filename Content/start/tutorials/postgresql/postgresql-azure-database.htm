Install drivers

CLI users

To use and PostgreSQL, you need the JDBC driver JAR file (Maven download).

<dependency>
    <groupId>org.postgresql</groupId>
    <artifactId>postgresql</artifactId>
    <version>42.6.0</version>
</dependency>

Also, you can use the PostgreSQL Extension JAR file, which is the vacuum extension that adds an additional tag or command to support vacuuming.

Flexible Server via Service Principal Name

If you use Azure Flexible Server for PostgreSQL and want to authenticate using Service Principal Name (SPN) in your automation pipeline, follow these steps to connect to :

  1. Create an automation script that requests these parameters from the user:
    • CLIENT_ID (username, example: 7ca151a7-cb01-4580-be03-0408df979c8f)
    • CLIENT_SECRET (password)
    • AZURE_TENANT_ID (the Active Directory tenant GUID to authenticate; a constant in your environment)
  2. Run the following commands in automation:
    3. az login -u "$CLIENT_ID" -p "$CLIENT_SECRET" -t "$AZURE_TENANT_ID" --service-principal

Troubleshooting

If you use Azure Database with PostgreSQL and the connection pooling manager PG Bouncer, you may receive this error when you try to use :

prepared statement ... does not exist

If you receive this error, it is because you are using the connection port 6432, which is typical for PG Bouncer. However, for 's prepared statements to work, the connection pooling mode must be set to session. However, using Azure with PostgreSQL does not support the session value. Therefore, 's prepared statements may fail, causing the execution to fail.

To resolve this error, set your connection port to 5432.
