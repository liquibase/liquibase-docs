Using JDBC URL in

A JDBC URL is a database connection URL that identifies a database so that the appropriate driver recognizes it and connects to it. requires a JDBC driver to establish a connection between and your database.

All JDBC URLs begin with jdbc:. The format of other string elements depends on the JDBC driver. In most cases, the format of the database connection URL resembles the following example:

jdbc:<database>://<host>:<port>/<database_name>;<URL_attributes>
database name Name of the database to which you want to connect. Can also be the file system path to the database.
URL attributes Supported s of the database connection URL, such as upgrade=true and create=true.
host Name or address of the machine on which the server is running.
port Port number that the server framework uses.

To use the JDBC URL, complete one of the following tasks:

  • Run the --url parameter in the CLI:

    • --url=jdbc:<database>://<host>:<port>/<database_name>;<URL_attributes>

  • Configure LIQUIBASE_COMMAND_URL as an environment variable to set it once and pass connection information to and your database. Environment variables are only available in .

    • LIQUIBASE_COMMAND_URL =jdbc:<database>://<host>:<port>/<database_name>;<URL_attributes>

  • In the properties file, specify a value for liquibase.command.url to set it once and pass connection information to and your database.

    • url: jdbc:<database>://<host>:<port>/<database_name>;<URL_attributes>

Command-line arguments override environment variables and the properties file data. Environment variables override the properties file data. The properties file does not override any values.

For a list of JDBC examples for different databases, see [%=General.Liquibase%] Database Tutorials.
