Developer Workflow

Let's go through the entire developer workflow from beginning to end!

Step One

On your local development machine, add new s to your . The changes are your choice to make.

Step Two

Run liquibase update to apply the changes to the developer database. Iterate until you are happy with the final state.

Step Three

Run liquibase status --verbose --url=jdbc:h2:tcp://localhost:9090/mem:integration

Running the command in step 3 allows you to see that the integration database has s that were not applied.

It is always best practice to know what changes are going to take place before saving and applying the , especially if someone else edited or created the .

Step Four

Run liquibase update-sql --url=jdbc:h2:tcp://localhost:9090/mem:integration

Running the command in step 4 allows you to see exactly what SQL would run when running the update command. You can also run a diff command to compare the changes before saving and applying them.

To run the diff command, run: liquibase diff --url=jdbc:h2:tcp://localhost:9090/mem:dev --referenceUrl=jdbc:h2:tcp://localhost:9090/mem:integration --referenceUsername=dbuser --referencePassword=letmein

Step Five

Save your to source control, when you are happy with the changes that will be applied.

Step Six

Run liquibase update --url=jdbc:h2:tcp://localhost:9090/mem:integration to apply the new s.

You can run the status, update-sql, and diff commands again as way to see that all the changes have now been applied to integration.

DONE!

That's the whole workflow! Now, start back at step 1 to continue iterating through the entire cycle until you have a good feel for how works.

What Now?

Now that you have a feel for against your sample database, you can try it against your standard databases.

Just add your database's jdbc drivers to the lib directory in the install directory and update the url, username, and password fields in the properties file.

4.6.1 and later versions come with pre-installed drivers. For more information, see Adding and Updating [%=General.Liquibase%] Drivers.

Also, check the documentation associated with your specific database to locate the correct url format and download the driver jar. Then use the Developer Workflow to make changes against your own database.
