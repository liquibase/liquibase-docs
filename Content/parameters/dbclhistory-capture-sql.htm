dbclhistory-capture-sql

--dbclhistory-capture-sql is a global Boolean parameter that specifies whether captures SQL generated by commands and stores in the [%=General.databasechangeloghistory%] table. It is available in 4.26.0 and later. The default value is true.

Uses

When you run a command like update or rollback, uses that information to construct a SQL query that it sends to your database. By default, records all SQL these commands generate in the (DBCLH) table.

However, if you want to exclude executed SQL from your DBCLH, you can set --dbclhistory-capture-sql to false.
