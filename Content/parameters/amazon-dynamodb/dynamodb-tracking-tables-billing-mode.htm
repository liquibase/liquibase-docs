dynamodb-tracking-tables-billing-mode

--dynamodb-tracking-tables-billing-mode is a global parameter for the DynamoDB Pro extension. It specifies the initial billing mode for  tracking tables: and , and . This controls how you pay for database read/write operations and capacity. Valid values are PROVISIONED and PAY_PER_REQUEST. The default value is PAY_PER_REQUEST.

Uses

The --dynamodb-tracking-tables-billing-mode parameter lets you define which billing mode DynamoDB uses for read/write operations makes to those tracking tables:

  • If you can't estimate your database's expected load on the tracking tables in advance, use PAY_PER_REQUEST. This way, you won't reach capacity.
  • If you can estimate your database's expected load on the tracking tables in advance, use PROVISIONED. Then set custom capacity values for the following parameters:
    • --dynamodb-tracking-tables-provisioned-throughput-read-capacity-units
    • --dynamodb-tracking-tables-provisioned-throughput-write-capacity-units.
