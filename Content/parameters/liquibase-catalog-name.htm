liquibase-catalog-name

The liquibase-catalog-name parameter is a string that specifies the catalog to use for objects. It allows to access a specific database catalog where can store the and tables. The concept of a catalog varies between databases because not all databases have catalogs. For more information, refer to your database documentation.

Uses

A database catalog is typically an organized inventory of data assets in your database. It helps you collect and manage your metadata with the definitions of database objects, such as tables, views, synonyms, value ranges, indexes, users, and user groups.

You can use the liquibase-catalog-name parameter to specify the catalog in which to locate and store objects created when running operations.
