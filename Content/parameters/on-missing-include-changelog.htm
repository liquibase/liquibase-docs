on-missing-include-changelog

The on-missing-include-changelog global parameter controls how behaves when it cannot find a specified in an include tag in the root . It is available in 4.19.1+.

If you set on-missing-include-changelog to WARN and the included is not present, a warning message will appear in logs. If it is set to FAIL and the included is not present, returns an error. The default configuration is FAIL. We recommend setting it to WARN.

Uses

If your included was removed by accident, will notify you immediately so you know the exact cause of the error. With on-missing-include-changelog enabled, you will be able to quickly determine why the file is missing and how to restore it.

Output

If you set --on-missing-include-changelog to WARN and cannot find the you specify in <include>, your log output displays the following:
