output-file-encoding

The --output-file-encoding global parameter is a string that determines which character encoding to use when writing command outputs to external files. By default, it is set to UTF-8.

Uses

If you need to send the output of a command like generate-changelog, update-sql, or diff to a text file that does not use UTF-8, you must specify the correct character encoding. This ensures that all symbols in your file display correctly.

Alternative or legacy character encodings include US-ASCII, Windows-1252, UTF-16, and others.
