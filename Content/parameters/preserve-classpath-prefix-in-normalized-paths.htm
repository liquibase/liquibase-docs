preserve-classpath-prefix-in-normalized-paths

The --preserve-classpath-prefix-in-normalized-paths global parameter is a Boolean value. It forces to preserve the prefix classpath: while it normalizes the resource path you have specified in --classpath. It is available in 4.30.0 and later.

The default value of --preserve-classpath-prefix-in-normalized-paths is false.

This parameter only affects resources you specify in the parameter --classpath. It does not affect any resources you specify in --search-path.

Uses

uses the Java Virtual Machine (JVM) to resolve resource paths like your classpath, which is a list of file paths where s, configuration files, and JAR files (bundles of Java classes) are stored. The JVM uses the classpath to search for, load into memory, and link each of those resources to prepare them for use in .

lets you set the classpath using the global parameter --classpath. normalizes the classpath value you specify by converting it to a consistent format. By default, if your classpath contains the prefix classpath:, removes the prefix. Then sends the normalized classpath to the ResourceAccessor.

However, you may be trying to implement a custom ResourceAccessor that needs to accept an abstract URL as the classpath. For example, you may want to specify a root resource path containing a nested (hierarchical) classpath as the value of --classpath instead of a relative or absolute path directly connected to the file system. If strips the classpath: protocol, your custom class cannot access the correct sub-resource locations.

If you want to preserve the classpath: prefix in your resource path, set --preserve-classpath-prefix-in-normalized-paths to true. This way, path normalization does not strip classpath: from the resource path it sends to the ResourceAccessor.
