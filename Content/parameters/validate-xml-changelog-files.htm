validate-xml-changelog-files

validate-xml-changelog-files is a Boolean global parameter. It specifies whether performs XSD validation of XML files. The default value is true.

Uses

publishes XSD (XML Schema Definition) files with every release. The XSD specifies how your XML must be formatted in order to be considered valid. By default, automatically validates your against the standard defined in the XSD for your version of .

If you include many XML s in your root , the validation process may take a long time. If you know that your matches the XSD and doesn't require validation, you can set validate-xml-changelog-files to false to disable the validation process.

If your workflow requires you to use non-standard syntax in your which fails the XSD validation despite functioning correctly, it is possible to bypass the validation by setting this parameter to false. However, you can alternatively create a custom XSD for your use-case. For more information, see Example Changelogs: XML Format.
