pro-global-end-delimiter

--pro-global-end-delimiter is a global argument that lets you specify a default delimiter to use for all s. This is useful if you can't use the default delimiter of ;. It is available in 4.25.1 and later and requires a valid license key to use.

The --pro-global-end-delimiter argument is similar to the endDelimiter  for the <sql> , but has a global scope. If you set --pro-global-end-delimiter and do not set endDelimiter, then applies the value of --pro-global-end-delimiter to all s. If you set a delimiter both ways, determines which delimiter to use for a in the following order:

  1. endDelimiter set on a specific  (highest precedence)
  2. endDelimiter set in a called by include or includeAll in modifyChangeSets
  3. --pro-global-end-delimiter set globally (lowest precedence)

You can give --pro-global-end-delimiter the highest precedence of all delimiter settings by also using the --pro-global-end-delimiter-prioritized parameter.

Uses

If you need to deploy SQL using a non-standard delimiter, such as if the SQL you're deploying contains the ; character, you can use --pro-global-end-delimiter set a custom delimiter across all your s. This is a much faster way to set a delimiter than manually specifying endDelimiter on every individual . Also, having one central argument for a delimiter reduces inconsistencies resulting from manual updates to s.
