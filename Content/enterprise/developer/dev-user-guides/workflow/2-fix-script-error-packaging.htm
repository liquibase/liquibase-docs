2.0. Fix a script error found during packaging

A packaging job can fail for many reasons:

  • Errors in the script
  • Changes already exist into the target database - maybe because someone deployed those changes manually
  • Dependent changes don't yet exist in the target database
    • Review the Processing Order for how Datical will determine in which order to deploy scripts

It is important to understand which stage of the workflow your error occurred in. The console output of the packager job can help you distinguish each stage

  1. Packaging stage
  2. Forecast stage
  3. Deploy stage

Review these troubleshooting documents to assess how to fix your errors:

  • Troubleshooting: Environment Issues
  • Troubleshooting: Remediation Process - Issues with Packaging
  • Troubleshooting: Remediation Process - Issues with Forecast or Deploy

To address non-environmental errors, please review your scripts you checked in earlier and apply necessary fixes. Then check-in your changes to SQL repo again. This will trigger the Datical packaging job again.
