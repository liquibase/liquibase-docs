7.0. Abandon a change (and clean up)

  1. use -cleanup

Stored Logic folders (procedure, function, package, packagebody, trigger, view)

  • If you already have a current version of a stored logic script (such as version 5) but now you want to use an older version of the script (such as version 2), you can edit the stored logic script again to create a new higher version (such as version 6) that has identical contents to the old desired version (such as version 2). Edit the script to have the desired contents, commit it to source control, then run packager.
  • If you have a use case where you deployed a stored logic script which is no longer needed, you can package a script to drop the object.

ddl,ddl_direct, sql, data_dmlandsql_directfolders

If you want to cleanup changes from a particular script:

  • Check in a script with cleanup or rollback code and name the file with -cleanup in the file name
    • For example, if you want to clean up changes from scripts "member_siblings_tab.sql", then check in the cleanup script with this name "member_siblings_tab-cleanup.sql"
    • Note: In Datical changeset, this will also set the "ignore" attribute to "true"
