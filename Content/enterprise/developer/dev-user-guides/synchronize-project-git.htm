User Guide: Synchronize Project (Git)

This documentation shows you how to synchronize your project from GUI.

Prerequisites

It is assumed that your project is already connect to a Git repository and that an initial commit was previously performed. See Implementation Guide: Initial Commit of project to Git for more information on connecting your project to Git repository.

Overview

There are two scenarios that are possible here:

  1. Someone (or an automated process such as deploy packager) made changes to project and the user would like to obtain those changes from Git.
    1. This is a "git pull" use case
    2. You may not know whether a person or process has made recent changes, so it is best practice to do "git pull" to make sure you have the latest before working on and/or pushing changes to a project.
  2. User makes changes to project and would like to commit those changes into Git
    1. This is a "git push" use case

It is a best practice to always perform a "git pull" prior to doing a "git push".

Even if you didn't make any previous changes from the Datical GUI, an automation job may have made changes to the project by way of running a Datical Packager job via command line and pushed changes into Git repo. Therefore, always perform a "git pull" first before a "git push" from Datical GUI.

Obtain Changes from Git – "git pull"

  • Right-click on the project → Team → Pull
  • If there are changes found to your project on Git repository, then those are pulled in and your project will be updated with those changes. If there are no changes, then
  • If you get a message that says "Nothing to update - everything up to date", then that means you already have the latest changes available to you.

Commit Changes into Git – "git push"

After you have done a "git pull" and then worked on a project, you may have made new changes that need to be committed and pushed to the git repository. When you have made changes to project you can commit those changes into the Git repository using built in capabilities implemented in the GUI.

  1. From GUI, right-click on the project → Team → Commit
  2. A new Git Staging view will appear. You will be asked to stage files (if not already in Staged Changes box) and provide comments for the commit. If there are no file in Staged Changes box, you will need to move files from Unstaged Changes using the highlighted button.
  3. Press Commit and Push.

  5. Click Close.
