How To: Import an Azure DevOps Git repository with a Project using the GUI

Step-by-step guide

Clone your Azure DevOps repository

  1.  Open the GUI, Click File → Import → Team → Click Project from Team Foundation Server → Click Next

2. In the Server Selection window select "Connect to a Team Services account" and Click the "Servers..." button.

3. In the Add/Remove Team Foundation Server window Click the "Add..." button → Paste your Azure DevOps Git Repository URL under "Name or URL of Team Foundation Server"

4. Click the "Enter Credentials..." button → in the sever authentication window under Credential type: choose "Username and password" → enter Username and Password → Click OK

5. In the "Add Team Foundation Server window click OK

You should now see a new Server listed in the "Add/Remove Team Foundation Server" window

6. Click close button to close the "Add/Remove Team Foundation Server" window

a. In the Import Projects from Team Foundation Server click next

b. You should see your new Project and https url Server appear under "Team Project Selection" → click to choose your project and hit Next

c. In the TFS Workspace Select your workspace → click Next → under Import Project(s) select your project → click Next → click Finish

7. In the Team Explorer under "Git Repositories" you should notice the new repositories.

a. Right click the project repository → Import Repository

b. In the Repository Selection window, select your Project Repository → click Next → Select the path where you want the repository to be in your local file system → Next → Finish

8. You should now notice that your project appears under the "Deployment Plans" Tab and ready to be used with Git commands.
