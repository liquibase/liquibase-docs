Import Projects

Considerations for Project Imports

You may import existing projects into a work space using the Import Wizard in the GUI. Here are a few best practices to ensure the import is successful:

  • Remove Old, Unneeded Files from Your Project - Over time, you may accumulate a large quantity of reports and log files for which you no longer need ready access.

    Reports and logs may be safely deleted, but removing items from the Resources directory can lead to operation failures. We recommend that you DO NOT alter items in the Resources directory.

  • Copy the Project Directory to the Datical Workspace Before Performing the Import -  There are known issues in the Eclipse framework that could lead to partial project imports if you select the "Copy projects into workspace" option when importing a large project with many files. To ensure that the entire project is available in the workspace, we recommend that the project directory be copied into the workspace manually before the import is performed for large projects. By default the Datical workspace is a directory named datical in the user's home directory.

Project Import Instructions

The following example illustrates the steps you need to take to import a project.

  1. Start the GUI.
  2. Select File > Import.

  3. Select Existing Deployment Projects and click Next

  4. You can import multiple projects in the same operation by putting several Datical projects in a common directory or in a zip archive. Depending on the method you employ, select the appropriate option and then click the Browse button to select select your directory or archive.

  5. Projects in the directory or the archive should now be listed in the middle Projects panel. Select the projects you wish to import and then click Finish.

  6. The projects are imported and you will see the new projects in the Deployment Plans list (left panel) of .

  7. After you have imported the project(s), when you are ready to close the GUI it is important to use the red X option. Close the GUI using the red X in the top right corner after importing projects. (Do NOT use the "File" then "Exit" option in the top left corner as it may cause errors upon reopening.)
