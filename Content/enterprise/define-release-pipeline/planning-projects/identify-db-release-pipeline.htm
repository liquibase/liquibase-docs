Identifying Databases for the Release Pipeline

When you create a project, you provide connection information for the following databases:

  • REF database, a schema-only database (no or minimal data). You need to create it to be used by to stage changesets produced by packaging SQL scripts. 
  • Each managed database in the release pipeline. There may be one or more databases per stage, which represent functional areas in the release pipeline. Typical stages in a release pipeline are DEV, QA, STAGE, and PROD. 
    • DEV - Develop application and database schema changes
    • QA - Formal testing
    • STAGE - Final testing and preparation for release
    • PROD - Preparation for release and release
  • Deployment Monitoring Console database (DMCDB), a database used by all projects to record Datical operations

Have the connection information available when you create projects.
