Update DMC

Version Compatibility

If you use DMC (Deployment Monitoring Console), you may need to update DMC when you update your version of .

To find which versions of DMC can be used with which versions of , expand the and Deployment Monitoring Console Compatibility Matrix section at DMC Compatibility Matrix and Release Notes.

DMC Upgrade Instructions

If the Java version was manually changed, you will need to restore DMC's default version of Java (1.11.8) before upgrading DMC. Before upgrading DMC after the Java upgrade, move or copy the previous version of Java (1.11.8) from the /opt/datical/old_jre directory back to the /opt/datical/jre directory prior to upgrading your DMC. DMC expects Java version 1.11.8 to be in the /opt/datical/jre directory for the DMC upgrade process to be successful. After your DMC upgrade is complete, you can change the Java version to a different 11.x java version again. For more information, see Change the Java version for DMC.

  1. Log in to https://download.liquibase.com. Go to Folders, Common, Datical_DMC_Software, and download the .sh file for the version of DMC that is compatible with your version of Liquibase Enterprise.
  2. Copy the .sh file to your existing DMC server.
    • Example: datical-service-unix-7_17_243-master.sh
  3. Run the installer with the following command.
    • Put in the appropriate version number:
      • /home/datical> sudo bash /datical-service-unix-<version>-master.sh
    • Example:
      • /home/datical> sudo bash /datical-service-unix-7_12_216-master.sh
  4. Enter your existing OS user as the admin user and admin group.
  5. Enter the directory of the previous DMC installation when prompted for an install location.
  6. The installer will update your installation and restart affected DMC services. You can begin using the new version of the DMC immediately.
