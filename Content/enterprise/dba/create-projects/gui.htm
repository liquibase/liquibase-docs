Creating a Project Using the GUI

Create New Project (Deployment Plan)

  1. Right-click on the left pane and select New > Deployment Plan.
  2. Enter the Project name, then click Next.

  3. Enter the Name for the step, select theDatabase Platform, then clickNext

  4. Enter the database connection information - make sure to click Test Connection before clicking Next

  5. If you took the defaults, part of creating the project is performing the baseline snapshot. The baseline snapshot captures the current state of the database schema and stores it in Datical ChangeSets for later forecasting and modification.

Add Another Database to the Project (add a Step to the Deployment Plan)

One of the core benefits of using is the ability to deploy your ChangeSets to multiple environments in a pipeline.

  1. To set up an additional step in the Deployment Plan, click the Add Step After link



  2. Enter Name for the step, select theDatabase Platform, then click Next.

  3. Enter the database connection information. We strongly recommend to use runtime credentials because it is more secure than stored credentials (due to the possibility of the stored password being decoded). Click Test Connection. If the connection is successful, click Next


  4. The new step is now available in your project.

Delete a Database (Step) from the Deployment Plan

  1. On the Deployment Plan pane, locate the step you want to delete
  2. Click the Remove Step link below the appropriate step
  3. Click OK to complete the delete

  4. The step is now deleted
