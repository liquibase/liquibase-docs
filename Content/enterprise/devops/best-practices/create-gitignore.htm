Create a .gitignore file

While performing normal operations (Packager, Forecast, Deploy, etc.) will generate log files and reports that are not necessary to be pushed into Git.

It is normal Git practice to ignore certain file types, or certain directories, because over time those will unnecessarily consume disk space.

This is implemented using a .gitignore file. Therefore, files and directories generated as a result of running various tasks should be ignored. This document will help you setup a .gitignore file in your project.

Some files generated as part of Datical operations, such as HTML reports, are important to be archived. However, Git is your source code repository, not a means to archive reports. Consider archiving reports into a separate directory or use a different tool such as Artifactory.

  1. In your project directory, create a new text file and name it .gitignore.

    • In version 8.3 and later, ignore files are installed automatically, so you will not need to create one. It will already be available.

    • Note that there is no extension to this file, such as .txt
  2. Contents of this file should be as follows:
    1. .gitignore
      Logs/
Snapshots/
Profiles/
Reports/
.auditdb/
/.classpath
/.metadata
/.reporttemplates
/daticaldb.properties
/daticaldb*.log
/packager*.log
*.zip

  3. Now, when you push your files, Git will use the .gitignore file to identify which files to prevent from commiting/pushing in. Steps to commit/push your code into Git is described in this page: Implementation Guide: Initial Commit of Project into Git
