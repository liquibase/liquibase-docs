Create a .tfignore file

While performing normal operations (Packager, Forecast, Deploy, etc.) will generate log files and reports that are not necessary to be checked into TFS.

It is normal TFS practice to ignore certain file types, or certain directories, because over time those will unnecessarily consume disk space.

This is implemented using a ".tfignore" file. Therefore, files and directories generated as a result of running various tasks should be ignored. This document will help you setup a ".tfignore" file in your project.

Some files generated as part of Datical operations, such as HTML reports, are important to be archived. However, TFS is your source code repository, not a means to archive reports. Consider archiving reports into a separate directory or use a different tool such as Artifactory.

  1. In your project directory, create a new text file and name it ".tfignore".  

    • In version 8.3 and later, ignore files are installed automatically, so you will not need to create one. It will already be available.

    • Note that there is no extension to this file, such as ".txt"
  2. Contents of this file should be as follows:
    1. .tfignore
      \Logs\
\Snapshots\
\Profiles\
\Reports\
.classpath
.metadata
.reporttemplates
daticaldb.properties
daticaldb*.log
packager*.log
*.zip

  3. Now, when you check-in your files, TFS will use the ".tfignore" file to identify which files to prevent from checking in. Steps to check-in your code into TFS is described in this page: Implementation Guide: Initial Commit of Project into Team Foundation Server (TFS)
