Automating Operations

Although operations can be performed manually, the best practice for DevOps environments is to run operations from automation software.

Typically a workflow in an automation solution includes steps that run operations through the CLI.

  • Status
  • Forecast
  • Deploy
  • Rollback
  • (Any Command)

Packaging SQL Code into changesets is typically managed by automation in the SCM environment (a "hook" or "trigger" that runs the Deployment Package when changes are checked in).

Configuring Automation Solutions to Run Operations

An integration provides the mechanism (plug-in) for an automation solution to run CLI commands.

You typically need to do the following to set up automated operations:

  1. Install the plug-in in your automation solution.
  2. Set global properties that store information about how to run the commands:
    • Location of the hammer command
    • Location of Datical database drivers
  3. Create steps in the automation solution to perform Datical operations. Each step runs one or more Datical Commands.
