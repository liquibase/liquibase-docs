Variation: Using the GUI to Install and Drivers on Linux Systems

Use Case: Individual User

The scenario is for a user system that has access to the internet.

  1. Download the installer for Linux (.jar).
  2. Run the GUI installation to install the clients (GUI and CLI).
  3. Start the GUI.
  4. Configure the GUI to add Update Site repositories.
  5. Install the JDBC drivers.
  6. Install the SCM plugins (optional).

Prerequisites

You have downloaded the installer for Linux systems

  • Download the following files from download.liquibase.com:
    • Linux installer (.jar file) - Example: DaticalDB-linux.gtk.x86_64-5.12.5833
    • Composite repository - Example: DaticalDBCompositeRepo-5.12.5833.zip
  • See Downloading the Liquibase Enterprise Installer for more details

Verify the JRE Version on the Installation Host is 1.11

  • Run the following "java -version" command in a terminal window to check that you are running a supported version of the JRE (version 1.11.x)

    # java -version
java version "1.11.0_xx"Java(TM) SE Runtime Environment (build 1.11.0_xxx)
Java HotSpot(TM) Client VM (build 24.75-b04, mixed mode, sharing)

Update the PATH Variable

  1. Add CLI to the path

    vi ~/.bash_profile
export DATICAL_DB_HOME=/opt/datical/DaticalDB
export PATH=${PATH}:${ DATICAL_DB_HOME }/repl

  2. Source the updated .bash_profile

    . ~/.bash_profile

Next, we will install the drivers for the command line.

Installing Drivers

Online Driver Installation for Command Line

If your Linux machine has access to the internet, at the command line run the following command:

<datical_install_dir>/repl/hammer installDrivers http://update.liquibase.com/latest/thirdparty

hammer installDrivers installs both the CLI and GUI drivers.

Offline Drive Installation for Command Line

Or, if your Linux machine does not have access to the internet you will need to download Composite Repo zip file corresponding to your version of (from http://download.liquibase.com). Then, at the command line, run the following command:

<datical_install_dir>/repl/hammer installDrivers jar:file:/<path_to_repo>/DaticalDBCompositeRepo-<version>.zip\!/

hammer installDrivers installs both the CLI and GUI drivers.

You need to replace "<version>" with your actual version number in the command above, and replace "<path to repo> with the actual path to your zip file.

Also note the escaped exclamation point in the command for Linux:

\!/

Drivers installation for GUI

Adding Installation Repositories Using the GUI

  1. Start the GUI
  2. In the window, selectHelp > Install New Software.

  3. You see theAvailable Software window.

  4. To the right of the Work with field, clickAdd. You see the Add Repository dialog.

  5. Fill in the the Name and Location values as follows for the third party repository, then clickAdd.

    Name:  ThirdParty Repository
Location: http://update.liquibase.com/latest/thirdparty

  6. Check the list of repositories. In theAvailable Software window, click the selection arrow in the Work with field. You should see the following list of available sites:

    --All Available Sites--
 ThirdParty Repo - http://update.liquibase.com/latest/thirdparty

    Installing the Database Drivers in the GUI

    1. Start the GUI
      • If you see theNew Deployment Project dialog, clickCancel to dismiss it.
    2. In the window, selectHelp > Install New Software. You see theAvailable Software window.

    3. In the Work with field, type or select http://update.liquibase.com/latest/thirdparty

    4. After tabbing out of the Work with field you should see a list of options:

    5. Select Drivers, then clickNext.
      1. If you are using Git for your source control, then Drivers should be theonlycheckbox that you enable. Do NOT enable any of the other options (such as CollabNet, SVN, GUI, REPL, eReview, JNA, Subclipse, Team Explorer). The plugin for Git is already included in your installation, so Drivers is the only item you need to select and install if your source control is Git.

    6. Install Details window. Inspect the list, then clickNext.
    7. Review License window. Select I accept to accept the licenses, then clickNext.
      • If you receive an unsigned content warning, clickInstall anyway to allow it.

    8. When installation is complete, You are prompted to restart . ClickRestart Now.

      • If you do not restart now, the changes take effect the next time you start .

Verify the Install

Run the following commands to verify the install. 

hammer checkdrivers
hammer show version
hammer show license
