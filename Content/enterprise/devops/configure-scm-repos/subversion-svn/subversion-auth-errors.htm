Subversion Authentication Errors

URL access forbidden for unknown reason

Problem You see the following Subversion authentication error message

  • URL access forbidden for unknown reason

Solution

  1. Make sure the URL starts with https rather than http.
  2. Your authentication system may be case sensitive for the user name. For example, changing the user name all caps to lowercase may resolve the issue.
  3. Clear the password cache and re-enter your password in the GUI.


To clear the password cache, delete the appropriate file in the Subversion runtime configuration area (for JavaHL)

  • Windows: %APPDATA%\Subversion\auth\svn.simple
  • On Linux and OSX: ~/.subversion/auth/svn.simple

If you connect to multiple repositories, you may need to search the cache (text) files. Delete the one containing the URL of the repository.
