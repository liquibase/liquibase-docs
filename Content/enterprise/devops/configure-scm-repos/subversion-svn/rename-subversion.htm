Rename a Project in Subversion

  1. Launch
  2. Make sure svn:ignore is not set to ignore .project or datical.project files
  3. Right click on project and select Team > Update to HEAD
  4. Right click on project and select Team > Commit
    • Enter a comment for the comment for the commit operation
  5. Backup Project Folder
    • Copy the project directory to a temporary directory
  6. Backup Project Folder
  7. Delete project from console
    • Select Delete project contents on disk (cannot be undone)
  8. Click the icon and Select Other ïƒ¨ SVN Repository Exploring
    • Right-click the Project to rename, then select Rename/move

  9. Expand the url and select the destination parent folder for your project.
    • Enter the new project name in Resource Destination name, then click Next.

    • Enter a commit comment, then click Finish.

  10. In SVN Repository Exploring right-click the renamed project, then select Checkout.
    • Select Check out as a project in the workspace.
    • Enter the new project name in Project Name, then click Next
    • Select Use default workspace location and confirm that the location is correct, then click Finish.

  11. Navigate to the Deployment tab and verify that the project was added with the new name
  12. In the Datical project directory, modify the .project & datical.project files with the new project name.



  13. Right-click the project, then select Team > Update to HEAD.
  14. Right-click the project, then select Team > Commit.
    • Enter a comment for the comment for the commit operation, then click OK.
  15. Right-click the project, then select Refresh.

The project is now ready to use.
