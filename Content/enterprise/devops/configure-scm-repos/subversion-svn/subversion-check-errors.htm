Subversion Check-In Errors

Working copy is not up-to-date

If you see the following out of date error when you try to commit your changes to Subversion, it means that there are changes in the repository that you need to pull before you can commit your changes.


Update your local project with the latest changes in the Repository

  1. In the Datical GUI, right-click the project, then select Team > Update to HEAD.

  2. The local project is updated with the latest changes from the repository.
