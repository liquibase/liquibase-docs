ignore

The ignore  is a Boolean that tells to treat a particular as if it does not exist. The default value is false.

Uses

The ignore  is useful if you want to make a database deployment, but want to exclude certain s from the deployment. For example, you may have a that should not be executed during an update, but you want to keep the in the for tracking purposes.

Syntax

SQL example 
-- liquibase formatted sql

-- changeset adrian:1 ignore:true
create table person (
    name varchar(255)
);
 XML example 


    <changeSet author="adrian" id="1" ignore="true">
        <createTable tableName="person">
            <column name="name" type="varchar(255)"/>
        </createTable>
    </changeSet>
</databaseChangeLog>
 YAML example 
databaseChangeLog:
    -  changeSet:
        author: adrian
        id: 1
        ignore: true
        changes:
        -  createTable:
            tableName: person
            columns:
            -  column:
                name: name
                type: varchar(255)
