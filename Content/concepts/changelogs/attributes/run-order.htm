runOrder

The runOrder  specifies whether a should be run before or after all other s instead of running it sequentially based on its order in the . Valid values are first and last. runOrder is not supported in formatted SQL s.

Uses

By default, when you run the update command, runs s sequentially according to their order in the . Changes at the top of the file are run first, and changes at the bottom are run later.

However, you may have a that you always want to run at the end of a deployment. Instead of moving it to the bottom of the for every database update, set runOrder to last on that . This way, it executes after all other s regardless of its order in the .

Syntax

XML example 
<changeSet  id="1"  author="your.name"  runOrder="last"  runAlways="true">
    <update  tableName="deployment_log">
        <column  name="latest"  type="DATETIME"/>
    </update>
</changeSet>
 YAML example 
databaseChangeLog:
  -  changeSet:  
      id:  1
      author:  your.name
      runOrder:  last
      runAlways:  true
      changes:
        -  update:
            tableName:  deployment_log
            columns:
              -  column:
                  name:  latest
 JSON example 
{
  "changeSet": {
    "id": "1",
    "author": "your.name",
    "runOrder": "last",
    "runAlways": "true",
    "changes": [
      {
        "update": {
          "tableName": "deployment_log",
          "columns": [
            {
              "column": {
                "name": "latest"
              }
            }
          ]
        }
      }
    ]
  }
}
