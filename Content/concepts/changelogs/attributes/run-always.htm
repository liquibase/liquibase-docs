runAlways

You can use the runAlways  to execute a every time you make a database deployment, even if it has been run before.

Uses

Normally, when you deploy a to your database with the update command, adds a row to the [%=General.databasechangelog%] table to keep track of it. In every later deployment, checks the table, sees that the has already been deployed, and skips it.

However, if you need a to repeat an action on every deployment, you can set runAlways to true for that . Possible workflows include:

  • Using update to specify the current timestamp in a table in your database
  • Using tagDatabase to tag the current state of your database
  • Using sql or sqlFile to run a script that dynamically updates object permissions

In these examples, you wouldn't necessarily change the s themselves in a deployment. However, you would want these s to run again because they behave based on how other parts of your database will have changed.

Setting runAlways to true on s that don't need to be run more than once may slow down your database updates.

If you want to run a on deployment only if it was updated since the last deployment, you should use runOnChange instead of runAlways.

Syntax

The default value for runAlways is false. Set runAlways to true on s that you want to execute on every migration.

SQL example 
-- changeset your.name:1 runAlways:true
UPDATE deployment_log SET latest=CURRENT_TIMESTAMP;
 XML example 
<changeSet  id="1"  author="your.name"  runAlways="true">
    <update  tableName="deployment_log">
        <column  name="latest"  type="DATETIME"/>
    </update>
</changeSet>
 YAML example 
databaseChangeLog:
  -  changeSet:  
      id:  1
      author:  your.name
      runAlways:  true
      changes:
        -  update:
            tableName:  deployment_log
            columns:
              -  column:
                  name:  latest
 JSON example 
{
  "changeSet": {
    "id": "1",
    "author": "your.name",
    "runAlways": "true",
    "changes": [
      {
        "update": {
          "tableName": "deployment_log",
          "columns": [
            {
              "column": {
                "name": "latest"
              }
            }
          ]
        }
      }
    ]
  }
}
