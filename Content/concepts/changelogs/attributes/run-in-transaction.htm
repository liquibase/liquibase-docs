runInTransaction

The runInTransaction  specifies whether a can be run as a single transaction (if possible). The default value is true.

Uses

When you make a database deployment with the update command, attempts to create an atomic "transaction block" around each to ensure that it is either fully deployed to the database or not deployed at all. This protects your database from incomplete deployments with unpredictable behavior.

However, your database may not allow certain SQL statements to run in a transaction block, such as CREATE DATABASE in PostgreSQL or ALTER TABLE on an external table in AWS Redshift. If you need to deploy a where this is the case, set runInTransaction to false on that .
