onValidationFail

onValidationFail is an you can specify in s in your . It controls what does when a fails validation. Values are HALT and MARK_RAN. Default: HALT.

onValidationFail does not relate to checksum validation. This is only valid for XML, JSON, and YAML s because does not validate the SQL within SQL s before it is executed.

Uses

performs basic validation on s you deploy, such as checking whether a change is valid for your database or whether you have specified all required s. If a fails this validation, by default halts the deployment and return an error message. For example:

Unexpected error running Liquibase: Error parsing line 40 column 65 of example-changelog.xml: cvc-complex-type.4: Attribute 'id' must appear on element 'changeSet'.

However, you can set onValidationFail to MARK_RAN if you want to mark a failing as run and continue with the deployment.

If your s are failing validation and you want a deployment to proceed, it is a best practice to fix the underlying cause of the failures instead of skipping them with onValidationFail.

Syntax

XML example 
<changeSet  id="1"  author="adrian"  onValidationFail="MARK_RAN">
    <createTable  tableName="company">
        <column  name="address"  type="varchar(255)"/>
    </createTable>
</changeSet>
 YAML example 
databaseChangeLog:
  -  changeSet:  
      id:  1
      author:  adrian
      onValidationFail:  MARK_RAN
      changes:
        -  createTable:
            tableName:  company
            columns:
              -  column:
                  name:  address
 JSON example

  "changeSet": {
