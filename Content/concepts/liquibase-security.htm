Collecting your data

only sends information to when you provide all of the following:

  • A valid Liquibase Hub API keys.
  • A registered file, except for the drop-all command which does not require a file.
  • The liquibase.hub.mode property set to all or meta. If you set the property to off, no data will be sent to .
  • Your system allowing https connections

Without all these conditions, no information will be sent to .

The information that sends to is determined by the liquibase.hub.mode property setting. recommends using LIQUIBASE_HUB_MODE environment variable to control which environments can send which kind of data. collects data based on the setting in the properties file or the Maven POM file:

  • off sends no data to , even if you have a valid API Key in your defaults file.
  • meta sends what considers the metadata, such as name, name, comments, username, timestamps, version, integration, and similar meta settings. Use the meta setting for connections where sensitive information might be transmitted using , typically DML scripts with personally identifiable information. Applying the meta setting prevents sensitive information from being transmitted to and limits usersâ€™ ability to use in identifying an update failure if one occurs.
  • all sends the metadata in addition to the body from the , generated SQL executed by the core app, operation-event logs, and change-event logs.
