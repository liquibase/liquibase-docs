dropPrimaryKey

The dropPrimaryKey drops an existing primary key in the table.

Uses

You can typically use the dropPrimaryKey when you want to remove a field that uniquely defines a record in your database. It can include tableName and other s listed in this documentation.

Running the dropPrimaryKey

To drop an existing primary key, follow these steps:

  1. Add the dropPrimaryKey to your with the needed s as it is shown in the examples.
  2. Deploy your by running the update command:
    3. liquibase update

Now, you should see that the information indicating your primary key is removed from the table.

Available s

Name Description Required for Supports Since
catalogName all
constraintName (primary key) to drop firebird, informix, sybase all
dropIndex all
schemaName all
tableName from which you drop the primary key. all all
XML example 
<changeSet author="liquibase-docs" id="dropPrimaryKey-example"> 
    <dropPrimaryKey constraintName="const_name"  
            dropIndex="true" 
            schemaName="public"
            tableName="person"/>
 </changeSet>
 YAML example 
-  changeSet:

    author: liquibase-docs
