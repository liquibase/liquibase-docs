createCompositeType

The createCompositeType defines a custom data structure that can be used in PostgreSQL tables, columns, and function signatures to organize related fields into a single logical structure.

Inspection features or commands, such as snapshot, diff, generate-changelog, and diff-changelog, are not currently supported with composite types.

Run createCompositeType

Available s

Name Type Description Requirement
catalogName String The name of the database catalog. Optional
schemaName String The name of the schema that contains the composite type you want to alter. Optional
typeName String The name of the type you want to create. Optional
typeAttribute String The individual fields inside of the composite type. This can be a column within a table. Optional

Examples

  • YAML example
  • JSON example
  • XML example 

databaseChangeLog:
  - changeSet:
      id: 1
      author: itTest
      changes:
        - createCompositeType:
            typeName: "myType"
            typeAttributes:
              - typeAttribute:
                  name: "attr0"
                  type: "int"
              - typeAttribute:
                  name: "Attr1"
                  type: "int"
              - typeAttribute:
                  name: "ATTR2"
                  type: "text"
                  collation: "en_US"
              - typeAttribute:
                  name: "att®3"
                  type: "line"
              - typeAttribute:
                  name: "attribute 4"
                  type: "real"

Database Support

This change type is only supported for PostgreSQL.
