A is an XML, YAML, or JSON formatted change that you can specify to update your database with . s correspond to SQL statements applied to your database, such as CREATE TABLE. You specify the you want to use within a Changeset in your Changelog. It is a best practice to include only one per .

This page lists all changes that you can apply to your database with the default installation. You can also write your own changes through the extension system.

For a guide on using s with , see Migrating with [%=General.Liquibase%] [%=General.changetypes%]s. If your s are written in SQL, see Migrating with SQL. In SQL s, database changes are written using raw SQL statements within s.

Notes

  • Property values are string unless otherwise noted.
  • Boolean parameters are defaulted to false unless otherwise noted.
  • Some change types automatically create rollback statements when you run rollback commands. For more information, see [%=General.Liquibase%] Rollback Workflow and Automatic and Custom Rollbacks.
