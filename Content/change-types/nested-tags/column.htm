column

The column tag specifies information about columns you create or modify with s like addColumn, createTable, createIndex, dropColumn, insert, loadData, loadUpdateData, and update. You can specify one or multiple columns in a single .

Uses

Use the column tag to define the behavior of your table columns. For example, with a like addColumn, setting the defaultValue  on a column specifies its default value. Setting the value  sets all rows existing to the specified value (such a string) without modifying the column default. Setting the valueComputed  as a function name or name of a computed column sets all rows to the value returned (computed) by that function.

You can use the column tag with createIndex to create an index on a particular table column (<column name="address"/>). You can also create an index on a computed version of that column (<column name="lower(address)" computed="true"/>).

Some that interact with table columns, like addPrimaryKey and modifyDataType, do not use the <column> tag. Instead, they use their own s like columnNames and columnName respectively to refer to existing columns specified with the <column> tag.

Available s

You can use the <column> tag with multiple s, but not every is valid for every . For example, if you use column with createTable, both name and value are valid s. However, if you use column with createIndex, name is valid but value is not.
