dropTrigger

The dropTrigger drops the trigger in the database.

Uses

You can typically use the dropTrigger  when you want to remove an existing trigger from your database. It can include catalog, schema, and table name s listed in this documentation.

Running the dropTrigger 

To drop a trigger in your database, follow these steps:

  1. Add the dropTrigger to your with the needed s as it is shown in the examples.
  2. Deploy your by running the update command:
    3. liquibase update

Now, you should see that the trigger you've specified is removed.

Available s

Name Description Required for Supports Since
catalogName db2, db2z, oracle, postgresql
schemaName all
scope all
tableName all

XML example 
<changeSet  author="liquibase-docs"  id="dropTrigger-example"> 
    <pro:dropTrigger  catalogName="department"
          schemaName="public"
          scope="A String"
          tableName="person"
          triggerName="A String"/>
</changeSet>
 YAML example 
-  changeSet:  
    id:  dropTrigger-example  
    author:  liquibase-docs  
    changes: 
    -  dropTrigger: 
        catalogName:  department 
        schemaName:  public  
        scope:  A String  
        tableName:  person  
        triggerName:  A String

-- changeset Liquibase Pro User:1-dropTrigger labels:dropTrigger
DROP TRIGGER [public].[A String]
