modifyChangeSets

The modifyChangeSets tag lets you include SQL scripts in your   while using an application's native executor—like PSQL, SQL Plus, or SQLCMD—instead of the JDBC default to deploy raw SQL scripts with a native executor. This removes the burden of converting raw SQL scripts into formatted SQL s. You can use it to specify an overarching executor for all files in your project. It is available in 4.18.0+.

Uses

If you have many s that need to be run with a native executor, and you organize your s in a "root" and "nested" format using the include and includeAll tags, you can use the runWith  (PSQL, SQLPlus, or SQLCMD) to specify an executor that applies to every in every file wrapped in the modifyChangeSets tag. Since your nested s inherit the executor specified in modifyChangeSets, you don't have to specify it on each individual you include.

include and includeAll tags are available only in structured s (XML, YAML, JSON). This is an existing limitation not introduced by modifyChangeSets.

If you want to make an exception for a particular you're including, you can specify the default JDBC executor for that . All other s still run with the executor inherited from modifyChangeSets.

You can still include additional files in your root outside of the modifyChangeSets tag. These files run with the JDBC executor.

Run modifyChangeSets

Syntax

s

Description Required for Supports
endDelimiter

In 4.25.1, you can use endDelimiter with modifyChangeSets to apply a delimiter to all s you reference with include and includeAll.

endDelimiter is similar to the --pro-global-end-delimiter argument, but has a local scope. If both arguments are set, they follow the following precedence:

  1. endDelimiter set on a specific  (highest precedence)
  2. endDelimiter set in a called by include or includeAll in modifyChangeSets
  3. --pro-global-end-delimiter set globally (lowest precedence)
 none all

Nested tags

Name Description Required for Supports
include Includes the specified none all
includeAll Includes the specified directory of s none all

Examples

  • XML example
  • YAML example
  • JSON example 


    <modifyChangeSets runWith="sqlplus" endDelimiter="@">
        <include file="my/nested/changelog.xml" />
        <includeAll path="release200/changelogs/sql" />
    </modifyChangeSets>

    <include file="sql/another.sql"/>
</databaseChangeLog>
databaseChangeLog:
- modifyChangeSets:
    runWith: sqlplus
    endDelimiter: @
    include:
      - file: my/nested/changelog.xml
    includeAll:
      - path: release200/changelogs/sql
- include:
    file: sql/another.sql
{
    "databaseChangeLog": [
        {
            "modifyChangeSets": {
                "runWith": "sqlplus",
                "endDelimiter": "@",
                "include": {
                    "file": "my/nested/changelog.xml"
                },
                "includeAll": {
                    "path": "release200/changelogs/sql"
                }
            },
            "include": {
                "file": "sql/another.sql"
            }
        }
    ]
}

Example: deploy a single raw SQL file

  • XML example
  • YAML example
  • JSON example 
<modifyChangeSets runWith="psql">
	<include file="my-psql-script.sql" relativeToChangelogFile="true"/>
</modifyChangeSets>
databaseChangeLog:
- modifyChangeSets:
  - runWith: psql
  - include:
	   file: my-psql-script.sql
       relativeToChangelogFile: true
{  
  "databaseChangeLog": [
  "modifyChangeSets":
    {
      "runWith": "psql",
      "include": {"file": "my-psql-script.psql"},
      "relativeToChangeLogFile": {"true"}
					
    }
 ]
}
