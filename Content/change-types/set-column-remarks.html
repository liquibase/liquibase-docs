setColumnRemarks

The setColumnRemarks sets remarks on a column.

Uses

You can typically use the setColumnRemarks when you want to alter the existing column by adding a comment to it.

Running the setColumnRemarks

To set a remark on your column, follow these steps:

  1. Add the setColumnRemarks to your with the needed s as it is shown in the examples.
  2. Deploy your by running the update command:
    3. liquibase update

Now, you should see the column with a remark.

Available s

Name Description Required for Supports Since
catalogName asany, db2, db2z, h2, mariadb, mysql, oracle, postgresql 3.0
columnName all all
remarks all all
schemaName all
tableName containing the column to set remarks on all all
columnDataType mysql   4.3 (for XML)
columnParentType Indicates the parent object type of the column we are setting remarks against. Valid values are VIEW and TABLE. Default: TABLE.      
XML example 
<changeSet author="liquibase-docs" id="setColumnRemarks-example">  
    <setColumnRemarks catalogName="department"  
            columnName="id"
            remarks="A String"
            schemaName="public"
            tableName="person"
            columnDataType="int"/>
</changeSet>
 YAML example 
-  changeSet: 
    id:  setColumnRemarks-example
    author:  liquibase-docs
    changes:
    -  setColumnRemarks:
        catalogName:  department
        columnName:  id
        remarks:  A String
        schemaName:  public
        tableName:  person
        columnDataType:  int

 SQL example

The SQL to set column remarks is platform-specific. You need to use the example depending on your database syntax.
