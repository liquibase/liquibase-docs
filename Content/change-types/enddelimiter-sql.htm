endDelimiter SQL

endDelimiter is an that lets you specify a delimiter to separate raw SQL statements in your s. When you run commands like update or rollback, first parses any raw SQL you want to execute. looks for the endDelimiter string in each and splits the SQL on the endDelimter boundary. This ensures that your database receives only one SQL statement at a time, even if your raw SQL contains many statements.

You can use endDelimiter in a individual containing the sql or sqlFile s. endDelimiter is valid in Formatted SQL, XML, YAML, and JSON s. The default value is ; (semicolon). Your custom delimiter can be a combination of one or more letters, symbols, and/or numbers, or the empty string.

If your SQL only contains one semicolon at the end of every statement (or only one statement with no semicolon), you don't have to specify a custom delimiter. However, if your SQL embeds stored logic containing one or more semicolons, incorrectly parses your SQL as multiple statements, causing an error. You can set a different delimiter to prevent from processing incomplete SQL statements.

Delimiter settings

This page describes setting endDelimiter on s. However, there are several ways to set delimiters:

  1. endDelimiter set on a specific  (highest precedence)
  2. endDelimiter set in a called by include or includeAll in modifyChangeSets (4.25.1+)
  3. --pro-global-end-delimiter set globally (lowest precedence) (4.25.1+)

You can give --pro-global-end-delimiter the highest precedence of all delimiter settings by also using the --pro-global-end-delimiter-prioritized parameter.

Also, when running SQL with the execute-sql command, you can specify a delimiter with the --delimiter argument.

You can also use the rollbackEndDelimiter  on sql and sqlFile s in your . This lets you specify a delimiter for rollback SQL. It is only valid in Formatted SQL s.

Uses

The following examples demonstrate use of the endDelimiters SQL attribute:

Use endDelimiter in SQL for PROCEDURE and FUNCTION

An endDelimiter can be used when the contains SQL to create a stored procedure or function that contains the default ';' end delimiter. To avoid incomplete statements being sent to the database, the must be marked to have a different endDelimiter.

Use endDelimiter for some DBMS systems to run multiple statements

The endDelimiter must be specified for some dbms systems to run multiple statements.

Syntax

Escaping an endDelimiter

When setting an endDelimiter, note that certain DBMS and operating systems require delimiter values to be escaped. For example, a $$ endDelimiter with mysql requires escaping as: endDelimiter="\$\$". A migration error indicates the delimiter that needs to be escaped would be:
