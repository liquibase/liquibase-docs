removeChangeSetProperty

The removeChangeSetProperty tag lets you remove column properties you previously specified in a . This avoids errors when you deploy your changes on a database that doesn't support those properties. It is available in 4.24.0+.

Uses

Some databases support column properties like afterColumn, beforeColumn, and position. You can specify these with the addColumn . For example, in an XML :

<changeSet id="addColumn" author="mallod">
    <addColumn tableName="myTable">
        <column name="new_column" type="varchar(50)" afterColumn="test_col">
        </column>
    </addColumn>
</changeSet>

However, if you try to deploy this change to a database that doesn't support these properties, returns an error.

To remove a property, add removeChangeSetProperty above your s. replaces the property you specify with NULL and adds the new column to the end of the table. Now you can deploy without any errors.

If your database supports additional column properties, you can still use removeChangeSetProperty to remove them.

Run removeChangeSetProperty

s

Name Description Required for Supports
change The to modify. Valid values: addColumn all all
dbms all all
remove The property from addColumn to remove. Valid values: beforeColumn, afterColumn, position. Only removes a singles property. To remove more than one property, use an additional removeChangeSetProperty statement. all all
