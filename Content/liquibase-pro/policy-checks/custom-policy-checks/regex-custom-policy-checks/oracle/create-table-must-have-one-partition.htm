CreateTableMustHavePartition

CreateTableMustHaveOnePartition is a custom policy check that requires every CREATE TABLE statement to also have at least one PARTITION included.

regex: (?is)(?=.*\b(create)\b)(?=.*\b(table)\b)(?!.*\b(partition)\b).*

Step-by-Step

These steps describe how to create the custom policy check. It does not exist by default in .

  1. Enter this command into the CLI: 
    liquibase checks customize --check-name=SqlUserDefinedPatternCheck
  2. Give your check a short name for easier identification. In this example we will title the check:
    CreateTableMustHavePartition

  3. Set the Severity to return a code of 0-4 when triggered.
    Options: 'INFO'=0, 'MINOR'=1, 'MAJOR'=2, 'CRITICAL'=3, 'BLOCKER'=4

  4. Set the SEARCH_STRING to this valid regular expression:
    (?is)(?=.*\b(create)\b)(?=.*\b(table)\b)(?!.*\b(partition)\b).*

  5. Set the MESSAGE for when a match for regular expression <SEARCH_STRING> is found in a Changeset:

    Example: Error! CREATE TABLE statement include PARTITION information.

  6. Set STRIP_COMMENTS to true if you want to remove the comments from the output.

    The regex custom policy check is created successfully.

Sample Passing Script

--changeset amalik:employee CREATE TABLE employee ( id NUMBER(5), name VARCHAR2(30), week_no NUMBER(2)) PARTITION BY HASH(id) PARTITIONS 4 STORE IN (data1, data2, data3, data4 );

Sample Failing Scripts

--changeset amalik:sales CREATE TABLE sales ( salesman_id NUMBER(5), salesman_name VARCHAR2(30), sales_amount NUMBER(10), week_no NUMBER(2) ); --changeset amalik:company CREATE TABLE COMPANY ( COMPANY_ID INT NOT NULL, BOOKING_DATE DATE NOT NULL, ROOMS_TAKEN INT DEFAULT 0, ;
